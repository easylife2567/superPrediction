# MiroFish 项目代码质量审查与问题排查计划书

## 1. 审查目标
对 MiroFish 项目（前后端）进行全面的代码质量审查，识别潜在的架构缺陷、安全漏洞、性能瓶颈及维护性问题，并提供具体的修复建议和验证方法。

## 2. 审查范围与维度

### 2.1 代码规范与一致性 (Code Consistency & Style)
- **后端 (Python/Flask)**:
  - 检查是否遵循 PEP 8 规范（命名、缩进、注释）。
  - 审查 `backend/app` 下的模块划分是否清晰。
  - 检查类型注解（Type Hints）的覆盖率和准确性。
- **前端 (React/Vite)**:
  - 检查组件划分是否合理，是否存在巨型组件。
  - 验证 ESLint 规则执行情况，检查未使用的变量、导入等。
  - 审查 React Hooks 的使用规范（依赖项数组完整性）。
  - 检查 CSS/样式文件的组织结构。

### 2.2 安全性 (Security)
- **身份验证与授权**:
  - 现状确认：目前似乎无鉴权机制，需评估风险。
  - 检查 API 端点是否暴露敏感数据。
- **输入验证**:
  - 审查文件上传功能 (`backend/app/utils/file_parser.py`) 是否有文件类型和大小限制。
  - 检查 API 参数校验（如 Pydantic 或手动校验）。
- **配置安全**:
  - 检查 `.env` 文件及环境变量的使用，确认敏感信息（API Keys）未硬编码在代码中。
  - 审查 CORS 配置 (`flask-cors`) 是否过于宽松。

### 2.3 性能与资源 (Performance & Resources)
- **前端性能**:
  - 评估 D3.js 图谱渲染在大数据量下的性能表现。
  - 检查 React 组件的重渲染问题（使用 `useMemo`, `useCallback` 优化）。
  - 审查静态资源加载策略。
- **后端性能**:
  - 评估文件系统存储（JSON/TXT）在并发读写下的性能与锁机制。
  - 检查大文件处理时的内存占用。
- **内存泄漏**:
  - 检查前端定时器、WebSocket 连接、D3 实例的清理逻辑 (`useEffect` cleanup)。
  - 检查后端长运行任务（如模拟运行）的资源释放。

### 2.4 数据与存储 (Data & Storage)
- **数据完整性**:
  - 审查基于文件的存储方案 (`ProjectManager`) 在异常中断下的数据一致性风险。
  - 评估 Zep Cloud 集成的稳定性与错误处理。
- **查询优化**:
  - 虽然无 SQL 数据库，但需检查文件读取和 JSON 解析的效率。

### 2.5 异常处理与日志 (Error Handling & Logging)
- **后端**:
  - 检查是否有全局异常捕获机制。
  - 审查日志记录 (`backend/app/utils/logger.py`) 是否包含足够的上下文（时间、级别、Traceback）。
- **前端**:
  - 检查 API 请求的错误捕获与用户提示。
  - 建议引入 Error Boundary 防止页面崩溃。

### 2.6 测试与质量保障 (Testing & QA)
- **单元测试**:
  - 现状：仅发现 `test_profile_format.py`。
  - 计划：识别核心业务逻辑（如 `graph_builder.py`, `simulation_runner.py`）缺失的测试用例。
- **依赖管理**:
  - 检查 `requirements.txt` 和 `package.json` 中的依赖版本是否过时或存在已知漏洞。

### 2.7 架构与设计 (Architecture & Design)
- **API 设计**:
  - 审查 RESTful 规范遵循情况。
  - 检查请求/响应数据结构的统一性。
- **并发与多线程**:
  - 审查 Flask 中的后台任务处理（如模拟运行）是否阻塞主线程。
  - 检查多线程/多进程使用的安全性。

## 3. 输出成果
审查完成后，将生成一份详细的《MiroFish 代码审计与优化报告》，包含：
1.  **问题清单**: 按优先级（P0-P3）排序。
2.  **具体位置**: 文件路径与代码行号。
3.  **问题描述**: 现象与潜在影响。
4.  **修复建议**:具体的代码修改方案或架构调整建议。
5.  **验证方法**: 如何重现问题及验证修复有效性。

## 4. 执行计划
1.  **阶段一：静态分析** (自动工具 + 人工走查) - 预计耗时：较短
2.  **阶段二：动态验证** (运行项目，模拟场景) - 预计耗时：中等
3.  **阶段三：报告生成** (汇总分析，编写建议) - 预计耗时：较短
